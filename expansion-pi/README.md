# expansion-pi

Diretório para documentar expansões e implementações planejadas para o `recall-pi` com base nas capacidades do Pi.

Objetivo:
- separar propostas por função/capacidade
- evitar concentrar tudo em um único documento grande
- facilitar evolução incremental, revisão e implementação por fases

Estrutura sugerida:
- um arquivo por tema/capacidade
- cada documento com:
  - problema
  - objetivo
  - superfícies do Pi envolvidas
  - fases de implementação
  - riscos/observações

Documentos atuais:
- `TURNS-SESSION-DIGEST.md` — proposta de governança por turns, digest de sessão e notificações sem depender apenas de compaction
